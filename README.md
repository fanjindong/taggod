# 标签之神

<p align="center">
  <img src="assets/logo.png" alt="标签之神 Logo" width="220">
</p>

`标签之神` 是一个 Chrome 标签页管理插件，只解决一个核心问题：

在大量已打开标签中快速找回页面，并把当前窗口里散乱的标签一键整理成清晰、稳定、可继续工作的分组结构。

当标签栏挤满小图标、同一个网站散在不同位置、重要工作流被临时页面打断时，先用搜索快速切回目标页面，或直接整理当前窗口。

## 一键理顺满屏标签

打开弹窗后，搜索框会自动聚焦。可以输入标题、网址、主域名或自定义分组名，在所有已打开窗口中查找标签页；没有输入时会展示最近使用页面。

搜索结果支持键盘快速切换：

- `↑` / `↓` 选择结果。
- `Enter` 跳转到选中的标签页。
- 跨窗口结果会先聚焦目标窗口，再激活目标标签，避免只切换后台窗口里的标签。

点击“整理当前窗口”，插件会把当前窗口里的页面按主域名归拢到一起，并尽量保持你已经习惯的标签顺序。

固定标签会保留在前面，同一网站的多个页面会靠在一起。整理后的标签栏更适合继续工作，也更容易判断哪些页面还能关掉。

分组规则是插件的核心能力：

- 达到分组阈值的主域名会创建 Chrome 原生标签分组。
- 真实分组会排在零散单标签页前面，避免整理后仍然被单标签打断。
- 优先分组会排在普通分组前面，适合把常用工作站点固定到更靠前的位置。
- 同一主域名内按子域名稳定排序，让同站点的不同产品保持可预期顺序。
- 可以在“高级管理”里新增自定义分组规则，一条规则对应一个目标分组名。
- 规则内可以配置条件组，选择“满足全部”或“满足任一”，用于表达 AND / OR 组合。
- 多个域名进入同一个分组时，可以在同一条规则里添加一个“满足任一”的子条件组。
- 规则阈值可以单独设置；留空使用全局阈值，界面会直接显示最终生效的阈值。
- 即使没有达到建组阈值，同一项目下的标签页在整理后也会相邻排列，阈值只决定是否创建 Chrome 原生分组。

默认快捷键：

| 系统 | 快捷键 |
| --- | --- |
| macOS | `Command+Shift+Y` |
| Windows 和 Linux | `Ctrl+Shift+Y` |

默认打开快捷键：

| 系统 | 快捷键 |
| --- | --- |
| macOS | `Command+Shift+L` |
| Windows 和 Linux | `Ctrl+Shift+L` |

如果快捷键和你已有的扩展冲突，可以在 Chrome 的 `chrome://extensions/shortcuts` 里调整。

## 安装方式

1. 下载或克隆本项目。
2. 打开 Chrome 的 `chrome://extensions`。
3. 开启右上角的“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择本项目目录。
6. 把“标签之神”固定到浏览器工具栏。

## 权限与隐私

`标签之神` 只使用标签管理所需权限：

- `tabs`：读取、激活、移动、创建和关闭标签页。
- `tabGroups`：创建和更新 Chrome 原生标签分组。
- `storage`：在本地保存插件配置。

所有数据保存在本地 `chrome.storage.local`，当前版本不包含账号系统，也不会上传你的标签页数据。

## 开发与校验

本项目不需要构建步骤，源码可以直接作为 Chrome 插件加载。

运行本地校验：

```bash
node scripts/校验插件.js
```

也可以分别检查脚本语法：

```bash
node --check background.js
node --check popup.js
```
