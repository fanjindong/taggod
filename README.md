# 标签之神

<img src="assets/logo.png" alt="标签之神 Logo" width="256">

`标签之神` 是一个 Chrome 标签页管理插件，面向经常同时打开几十到上百个标签页的用户。它用一个弹窗完成整理、搜索、去重、分组和工作集保存，尽量减少在标签栏里来回找页面的时间。

## 功能特性

- 打开弹窗即可搜索所有已打开窗口的标签页。
- 未输入搜索词时，默认展示跨窗口最近使用页面。
- 支持点击结果直接切换，也支持上下键选择后回车切换。
- 一键整理当前窗口标签页。
- 按主域名创建 Chrome 原生标签分组，多个子域名会归到同一分组。
- 默认单个标签的主域名不创建原生分组，减少标签栏噪音。
- 分组阈值可配置，保存后会立即重新梳理当前窗口分组。
- 分组标题会省略常见公共后缀，短名冲突时自动显示完整主域名。
- 自动尊重当前标签栏里的分组顺序。
- 星标重要分组，让它们整理后优先靠前。
- 智能识别重复标签，关闭前展示确认列表。
- 去重时支持忽略常见追踪参数，减少同一页面被重复打开的问题。
- 按标题、网址、主域名搜索当前窗口标签页。
- 保存当前窗口为工作集。
- 工作集支持重命名、删除、收藏、恢复到当前窗口和恢复到新窗口。
- 保存工作集后可选择关闭已保存标签，便于临时清场。
- 使用快捷键整理标签页或保存工作集。

## 快速切换

弹窗的首要入口是“快速切换”。打开弹窗后，搜索框会自动聚焦，用户可以直接输入标题、网址或主域名来查找已经打开的标签页。

未输入搜索词时，插件会展示所有窗口里的最近使用页面，并排除当前窗口正在看的标签页。这样做是为了覆盖“我只是想回到刚才页面”的场景，同时避免当前页面占用首屏位置。

弹窗搜索框下方会用轻量键位提示展示可用操作，用户不需要阅读说明文档也能知道结果项可以点击、上下键可以选择、回车可以跳转。

搜索结果支持两种切换方式：

- 点击任意结果项，直接切换到对应标签页。
- 使用上下键选择结果，再按回车切换。

搜索只覆盖已打开标签页，不搜索工作集。原因是当前入口面向“切换页面”，而不是恢复历史工作现场。

## 整理规则

点击“整理标签页”后，插件会按下面的规则重新排列当前窗口：

1. 固定标签保持在最前面。
2. 星标分组排在非星标分组前面。
3. 多个星标分组之间，按当前标签栏从左到右的顺序排列。
4. 非星标分组之间，也按当前标签栏从左到右的顺序排列。
5. 无法识别顺序时，按主域名排序兜底。
6. 同一个主域名内按完整子域名排序。
7. 完整子域名相同时，保持原来的左右顺序。

这意味着：如果你已经在 Chrome 标签栏里手动把某个分组拖到前面，直接点击“整理标签页”即可，插件会尊重当前顺序。

## 分组阈值

弹窗中的“分组阈值”用于控制同一主域名至少有多少个标签才创建 Chrome 原生分组。默认值为 `2`，原因是单个标签单独建组通常会增加标签栏噪音。

保存阈值后，插件会立即重新梳理当前窗口分组：不满足新阈值的旧分组会被取消，满足新阈值的主域名会被创建或保留为分组。

## 分组显示名

插件内部仍使用完整主域名作为分组标识，原因是 `foo.com` 和 `foo.net` 不能被错误合并。Chrome 标签栏和弹窗中会优先展示短分组名，例如 `google.com` 显示为 `google`，`example.com.cn` 显示为 `example`。

如果当前窗口里存在短名冲突，插件会回退显示完整主域名，例如同时存在 `foo.com` 和 `foo.net` 时，会分别显示为 `foo.com` 和 `foo.net`。

## 优先分组

弹窗中的“优先分组”区域只展示当前分组阈值下会创建 Chrome 原生分组的主域名。例如 `mail.google.com` 和 `docs.google.com` 会归到 `google.com`；如果某个主域名只有一个非固定标签且分组阈值为 `2`，它不会出现在这里。

- 点击空心星标，可以把该分组设为优先分组。
- 点击实心星标，可以取消该分组的优先状态。
- 星标只决定分组是否进入优先区，不决定多个星标分组之间的顺序。

## 智能去重

点击“智能去重”后，插件会先扫描当前窗口标签页，不会立即关闭任何标签。重复项分为两类：

- 完整网址重复。
- 忽略 `utm_source`、`utm_medium`、`utm_campaign`、`utm_content`、`utm_term`、`ref`、`fbclid`、`gclid` 等追踪参数后重复。

每组重复标签会默认保留固定标签、当前激活标签或最左侧标签。只有点击“关闭已勾选重复标签”后，插件才会关闭用户确认过的标签。

## 工作集

“工作集”用于保存一组以后可能继续使用的标签页。保存时可以命名，默认名称为当前时间。

- 点击“保存工作集”可以保存当前窗口所有标签。
- 保存成功后可以选择“关闭已保存标签”，插件只会关闭本次保存时记录的标签。
- 工作集可以恢复到当前窗口，也可以恢复到新窗口。
- 工作集可以重命名、删除、收藏或取消收藏。

## 快捷键

整理按钮下方会用轻量键位样式提示默认整理快捷键，便于首次使用时发现高频操作入口。

默认快捷键：

- macOS：`Command+Shift+L` 打开插件弹窗。
- macOS：`Command+Shift+Y` 整理当前窗口标签页。
- macOS：`Command+Shift+S` 保存当前窗口为工作集。
- Windows 和 Linux：使用 `Ctrl` 替代 `Command`。

## 本地安装

1. 下载或克隆本项目。
2. 打开 Chrome 的 `chrome://extensions`。
3. 开启右上角的“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择本项目目录。
6. 点击浏览器工具栏中的“标签之神”图标。

## 开发与校验

本项目目前不需要构建步骤，源码可以直接作为 Chrome 插件加载。

运行本地校验：

```bash
node scripts/校验插件.js
```

预期输出：

```text
插件文件校验通过
```

也可以分别检查脚本语法：

```bash
node --check background.js
node --check popup.js
```

## 自动化与发布

仓库包含两个 GitHub Actions workflow：

- `校验`：在 PR、`master` 推送和手动触发时运行插件校验。
- `发布`：在推送 `v*` tag 时校验插件、打包 zip，并创建 GitHub Release。

建议在 GitHub 仓库设置中保护 `master` 分支：

1. 开启必须通过 PR 合并。
2. 要求 `校验 / 插件校验` 通过后才能合并。
3. 禁止 force push，原因是发布 tag 依赖稳定的提交历史。
4. 开启自动合并，让通过校验和评审的 PR 自动进入 `master`。

发布前先更新 `manifest.json` 的 `version`，然后创建同版本 tag。发布 workflow 会校验 tag 与插件版本一致，避免错误版本被发布。

```bash
git tag v0.1.1
git push origin v0.1.1
```

## 权限说明

`标签之神` 只声明实现功能所需的 Chrome 权限：

- `tabs`：查询、激活、关闭、移动和创建标签页。
- `tabGroups`：创建和更新 Chrome 原生标签分组。
- `storage`：保存工作集、分组快照和用户配置。

工作集和配置保存在本地 `chrome.storage.local`，当前版本不包含云同步。
