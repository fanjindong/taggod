# 主域名分组实现计划

> **给自动化执行者：** 按复选框逐步执行，每一步完成后运行对应校验。所有注释、文档和提交信息必须使用中文。

**目标：** 将标签整理、概览、优先分组和会话保存中的分组标识从完整域名改为主域名，并让单标签主域名不创建原生分组。

**架构：** 继续以 `background.js` 的 `getDomainKey(url)` 作为唯一主域名分组入口。新增完整主机名函数只用于组内排序，不改变弹窗和会话中的 `groupKey`。建组时过滤只有一个标签的主域名，避免单页分组增加标签栏噪音。

**技术栈：** Manifest V3、原生 JavaScript、Chrome Extensions API、Node.js 本地校验脚本。

**文件：**

- 修改：`background.js`

在 `GROUP_COLORS` 下方加入 `COMMON_MULTI_PART_PUBLIC_SUFFIXES`，用于解释为什么 `example.com.cn` 不能归并为 `com.cn`。

- [ ] **步骤 2：新增主机名判断与归一函数**

新增 `isIpAddressHost(hostname)` 和 `getPrimaryDomainFromHostname(hostname)`。短主机名、IP 地址和无法拆分的主机名保持原值，因为它们没有可靠的主域名。

保持异常网址归入 `其他`，可解析网址改为调用 `getPrimaryDomainFromHostname(parsedUrl.hostname)`。

### 任务 5：单标签不建组，组内按子域名排序

**文件：**

- 修改：`background.js`
- 修改：`scripts/校验插件.js`
- 修改：`README.md`

- [ ] **步骤 1：新增完整主机名函数**

在 `background.js` 新增 `getHostnameKey(url)`，用于返回完整 `hostname`。无法解析的网址继续返回 `其他`，原因是它们没有稳定子域名排序键。

- [ ] **步骤 2：调整排序比较器**

在 `organizeTabs()` 的主域名相同分支中，先比较完整主机名，再按原始标签位置兜底。

- [ ] **步骤 3：过滤单标签分组**

在 `organizeTabs()` 和 `regroupRestoredTabs()` 创建 Chrome 原生分组前，如果某个 `groupKey` 只有一个标签，则跳过创建分组。

- [ ] **步骤 4：补充校验断言**

在校验脚本中验证 `getHostnameKey('https://mail.google.com')` 返回 `mail.google.com`，确保主域名归并和子域名排序键不会混用。

- [ ] **步骤 5：更新中文 README**

说明单个主域名标签不会创建原生分组，且同一主域名内按子域名排序。
