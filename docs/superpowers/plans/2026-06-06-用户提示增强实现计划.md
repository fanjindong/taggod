# 用户提示增强实现计划

> 面向代理执行者：必须使用子技能 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans`，按任务逐项实现。步骤使用复选框语法跟踪。

**目标：** 在插件弹窗中常驻展示关键操作提示，让用户直接知道点击结果、上下键、回车和整理快捷键等高频能力。

**架构：** 本次只调整弹窗静态结构、样式和说明文档，不改后台标签逻辑。`popup.html` 承载提示节点，`popup.css` 统一弱提示样式，`scripts/校验插件.js` 负责防止关键中文提示被误删，`README.md` 同步面向中文开发者的说明。

**技术栈：** Chrome Manifest V3、原生 HTML、原生 CSS、原生 JavaScript、Node.js 本地校验脚本。

---

## 文件结构

- 修改 `scripts/校验插件.js`：补充弹窗和 README 的提示文案断言，先形成失败校验。
- 修改 `popup.html`：在搜索框、整理入口、更多工具各小节增加短提示。
- 修改 `popup.css`：增加统一轻提示样式，并适配整理入口从按钮网格扩展为按钮区。
- 修改 `README.md`：同步说明弹窗内已经展示这些提示，避免文档和界面脱节。

### 任务 3：补充轻提示样式

**文件：**
- 修改：`popup.css`

- [x] **步骤 1：让统一轻提示继承弱文本颜色**

在 `popup.css` 中找到：

```css
.popup-header p,
.section-title span,
.status-text,
.tab-url,
.session-meta {
  color: var(--muted);
}
```

替换为：

```css
.popup-header p,
.section-title span,
.status-text,
.tab-url,
.session-meta,
.helper-text {
  color: var(--muted);
}
```

- [x] **步骤 2：增加轻提示基础样式**

在 `popup.css` 的 `.quick-search-box` 规则后加入：

```css
.helper-text {
  margin: 0;
  font-size: 12px;
  line-height: 1.45;
}

.search-helper-text {
  display: block;
}
```

这里统一行高，原因是中文、快捷键和箭头符号混排时，默认行高在不同系统上会显得不稳定。

- [x] **步骤 3：适配整理按钮区和小节提示间距**

在 `popup.css` 中找到：

```css
.quick-action-grid {
  grid-template-columns: minmax(0, 1fr) auto;
}
```

替换为：

```css
.quick-action-grid {
  grid-template-columns: minmax(0, 1fr) auto;
}

.quick-action-area {
  margin-top: 14px;
}

.quick-action-area .quick-action-grid {
  margin-top: 0;
}

.action-helper-text {
  margin-top: 6px;
}

.section-helper-text {
  margin-bottom: 8px;
}
```

这里保留原有按钮网格类，原因是 `quick-action-grid` 的列宽规则已经符合当前两个按钮布局，只需要把外层改成可容纳提示的区域。
