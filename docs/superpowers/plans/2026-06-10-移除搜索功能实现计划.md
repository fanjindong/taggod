# 移除搜索功能 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 彻底移除插件内搜索能力，让产品聚焦一键整理和分组规则。

**Architecture:** 弹窗只展示整理入口和工具区，后台状态只围绕当前窗口标签构建。删除搜索快照、最近访问记录和跨窗口激活链路，保留去重的全窗口概览。

**Tech Stack:** Chrome Manifest V3、原生 JavaScript、HTML、CSS、Node `vm` 校验脚本。

---

### Task 1: 移除后台搜索链路

**Files:**
- Modify: `/Users/fanjindong/Public/tabgod/background.js`
- Test: `/Users/fanjindong/Public/tabgod/scripts/校验插件.js`

- [ ] **Step 1: 删除只服务搜索的存储与函数**

移除 `STORAGE_KEYS.recentAccess`、`RECENT_ACCESS_LIMIT`、`normalizeRecentAccessMap()`、`recordRecentTabAccess()`、`removeRecentTabAccess()`、`buildWindowOrderMap()`、`getWindowLabel()`、`getTabLastAccessedAt()`、`buildSearchTabSnapshots()`、`activateTabAcrossWindows()`。

- [ ] **Step 2: 收窄 `getPopupState()`**

让 `getPopupState()` 只读取当前窗口标签、工作集和设置，返回 `groups`、`overview`、`sessions`、`settings`，不再返回搜索用 `tabs` 和 `currentWindowId`。

- [ ] **Step 3: 删除消息与事件入口**

移除 `activate-tab` 分支，移除 `chrome.tabs.onActivated` 和 `chrome.tabs.onRemoved` 中只维护最近访问记录的监听。
