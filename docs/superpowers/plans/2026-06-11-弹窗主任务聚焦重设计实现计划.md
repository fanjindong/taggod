# 弹窗主任务聚焦重设计实现计划

> **给自动化执行者：** 必须使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans` 按任务执行。本计划使用复选框记录进度，不自动提交代码。

**目标：** 将插件弹窗调整为“主任务聚焦”结构：首屏突出一键整理，并把智能去重、保存工作集作为次级常用操作，高级管理承载配置和列表。

**架构：** 继续沿用原生 HTML、CSS 和 JavaScript，不引入新依赖。`popup.html` 负责信息结构，`popup.css` 负责视觉层级，`popup.js` 只调整文案、展开状态和刷新后的焦点保持，后台消息协议不变。

**技术栈：** Chrome 扩展 Manifest V3、原生 DOM API、原生 CSS、Node.js 校验脚本。

const popupHtmlPath = path.join(rootDir, 'popup.html');

const popupCssPath = path.join(rootDir, 'popup.css');
const popupCssContent = fs.readFileSync(popupCssPath, 'utf8');

assert.ok(popupHtmlContent.includes('class="primary-action main-organize-action"'));
assert.ok(popupHtmlContent.includes('class="secondary-action-grid"'));
assert.ok(popupHtmlContent.includes('aria-label="高级管理"'));
assert.ok(popupHtmlContent.includes('>高级管理</button>'));
assert.ok(!popupHtmlContent.includes('<h2>管理操作</h2>'));
assert.strictEqual((popupHtmlContent.match(/id="scanDuplicatesButton"/g) || []).length, 1);
assert.strictEqual((popupHtmlContent.match(/id="saveWorkspaceButton"/g) || []).length, 1);
assert.ok(popupCssContent.includes('.main-organize-action'));
assert.ok(popupCssContent.includes('.secondary-action-grid'));
assert.ok(popupCssContent.includes('.management-toggle-button'));

assert.ok(!readmeContent.includes('“更多工具”里新增自定义分组规则'));
