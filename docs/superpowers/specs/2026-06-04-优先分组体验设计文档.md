# 优先分组体验设计文档

## 背景与目标

当前 `标签之神` 的一键整理规则是固定标签优先、其余标签按域名排序并创建 Chrome 原生标签分组。这个规则稳定，但无法表达用户对某些工作分组的长期偏好。

本次增量目标是：用户不需要手写域名列表，也能让几个重要分组在每次整理后稳定排在前面。

## 方案取舍

推荐方案是 `星标分组 + 显式顺序管理`。

- 星标分组：弹窗展示当前窗口的域名分组，用户点击星标后，该分组以后整理时排在普通分组前面。
- 显式顺序管理：用户在“高级管理”的优先分组列表里通过上移、下移调整多个优先分组的整理顺序。

备选方案一是 `自动识别当前顺序`。这个方案不需要额外控件，但用户每次都要先在 Chrome 标签栏里手动拖动分组，顺序意图不够稳定。

备选方案二是 `当前活跃分组自动靠前`。这个方案不需要配置，但排序会随着当前页面变化，稳定性不足，适合作为后续增强。

备选方案三是 `手写域名列表`。这个方案实现简单，但用户体验差，容易输错域名，也不适合频繁调整。

## 首版范围

本次增量包含以下能力：

- 在弹窗中增加“优先分组”区域。
- 展示当前窗口按域名聚合后的分组列表。
- 支持点击星标，让分组每次整理时排在普通分组前面。
- 支持取消星标，让分组回到普通排序。
- 支持对已星标的优先分组上移、下移，并在一键整理时按保存顺序排列。

本次增量不包含以下能力：

- 手动输入域名。
- 拖拽调整弹窗内分组顺序，原因是上移、下移已经能覆盖低频排序场景，且实现风险更小。
- 自动学习用户长期行为。
- 跨设备同步优先分组。

## 数据结构

优先分组配置保存到 `chrome.storage.local` 的用户配置中。

```js
{
  priorityGroups: [
    {
      groupKey: "github.com",
      title: "github.com",
      starredAt: 1780540200000,
      sortOrder: 0
    }
  ]
}
```

字段含义：

- `groupKey`：用于排序和匹配的域名标识。
- `title`：弹窗展示名称，默认与域名一致。
- `starredAt`：星标时间，用于后续排查和扩展，不直接参与排序。
- `sortOrder`：优先分组内的显式排序号。旧配置没有该字段时，按原数组顺序补齐，避免升级后改变用户已有星标结果。

## 整理规则

一键整理的新排序规则如下：

1. 固定标签仍然排在最前面。
2. 星标分组排在非星标分组前面。
3. 多个星标分组优先按 `sortOrder` 从小到大排列。
4. 没有星标的分组，也按整理前当前窗口从左到右的分组首次出现顺序排列。
5. 如果无法得到当前顺序，再按域名字母顺序兜底。
6. 同一个域名内保持原始左右顺序。
7. 无法解析的网址归入 `其他`，没有优先配置时排在普通分组中。

固定标签继续保持最高优先级，是因为固定标签通常代表用户长期入口，不应该被普通整理规则挤到后面。

显式顺序用于表达用户长期偏好；当前标签栏顺序只作为普通分组和异常数据的兜底依据，避免未配置排序时出现不可预测结果。

## 弹窗交互

弹窗在“高级管理”里展示“优先分组”区域，原因是星标和排序都属于低频配置动作，不应挤占首屏整理和搜索空间。

区域包含：

- 当前窗口分组列表。
- 每个分组一枚星标按钮。
- 已星标分组显示上移、下移按钮。
- 分组名称和标签数量。

用户点击星标后：

1. 弹窗发送 `toggle-priority-group` 消息给后台。
2. 后台更新 `settings.priorityGroups`。
3. 弹窗刷新分组列表。
4. 状态提示显示“已置顶分组”或“已取消置顶”。

用户点击上移或下移后：

1. 弹窗发送 `move-priority-group` 消息给后台，并传入目标分组键和移动方向。
2. 后台交换相邻优先分组的 `sortOrder`。
3. 弹窗刷新分组列表，状态提示显示“已调整优先分组顺序”。

## 后台接口

新增消息：

- `toggle-priority-group`：切换某个分组是否优先。
- `move-priority-group`：调整已星标优先分组的相对顺序。

`get-state` 返回新增字段：

- `groups`：当前窗口分组列表。
- `settings.priorityGroups`：已保存的优先分组配置。

## 异常处理

- 当前窗口没有可识别分组时，显示空状态。
- 用户星标 `其他` 分组时允许保存，因为浏览器内部页面也可能是用户常用入口。
- 保存失败时显示失败提示，不改变当前弹窗状态。

## 测试策略

- 验证弹窗可以展示当前窗口分组列表和标签数量。
- 验证点击星标后，刷新弹窗仍能看到星标状态。
- 验证一键整理后，星标分组排在普通分组前，多个星标分组按保存的 `sortOrder` 排列。
- 验证上移、下移后，刷新弹窗仍能看到调整后的优先顺序。
- 验证取消星标后，该分组回到普通域名排序。
- 验证 `node scripts/校验插件.js` 通过。

- 已明确新排序规则和边界条件。
- 已明确新增存储字段和后台消息。
