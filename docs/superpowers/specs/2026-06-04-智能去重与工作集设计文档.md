# 智能去重与工作集设计文档

## 背景

`标签之神` 当前已经支持整理、搜索、完整网址去重、主域名分组、优先分组、会话保存和恢复。随着 Chrome 原生垂直标签页逐步补齐“标签太多时看不清、选不准”的问题，本项目不再优先建设完整标签管理工作台，而是优先加强 Chrome 原生能力没有覆盖好的两个场景：

- 关闭重复标签前给出明确确认，减少误关风险。
- 把“会话”改造成更容易理解和长期复用的“工作集”。

第一版采用“弹窗内分区增强”方案：继续以 `popup.html` 作为唯一入口，不新增独立详情页。

## 目标

- 将“关闭重复”升级为“智能去重”，支持确认后关闭。
- 将“最近会话”统一改名为“工作集”。
- 支持工作集命名、重命名、删除、收藏、恢复到当前窗口和恢复到新窗口。
- 保存工作集后提供“关闭已保存标签”的后续动作，但不自动关闭。
- 保持弹窗轻量，避免把功能扩展成完整标签管理工作台。

## 非目标

- 不做独立管理详情页。
- 不做完整标签工作台。
- 不做跨窗口全量标签管理。
- 不做相似标题、相似路径等高误判去重。
- 不做工作集导入导出。
- 不做云同步。
- 不做工作集分类。
- 不做自动保存历史快照。
- 不做自动关闭标签，因为关闭标签属于高风险操作，必须由用户明确触发。

## 整体信息架构

弹窗继续作为唯一入口，界面从上到下保持以下区域：

1. 顶部概览：标签数、主域名数、重复数、分组数。
2. 常用操作：整理标签页、智能去重、保存工作集。
3. 优先分组：保持现有星标逻辑。
4. 当前标签：保持搜索、切换、关闭。
5. 工作集：替代“最近会话”，展示收藏和最近保存的工作集。
6. 状态提示：显示保存、关闭、恢复等操作结果。

智能去重和工作集管理不一次性展开全部复杂内容，而是在用户触发后展开局部确认区或轻量弹层。这样可以保留弹窗的快速入口属性，避免界面变成完整管理后台。

## 智能去重

### 重复识别规则

智能去重只识别两类重复：

1. 完整网址重复：标签页网址完全相同。
2. 忽略追踪参数后重复：移除常见追踪参数后，规范化网址相同。

第一版忽略以下追踪参数：

- `utm_source`
- `utm_medium`
- `utm_campaign`
- `utm_content`
- `utm_term`
- `ref`
- `fbclid`
- `gclid`

忽略参数时只删除查询参数，不修改协议、主机名、路径和哈希。这样做是为了减少误判：有些页面会用路径或哈希表达真实内容状态，不能随意合并。

### 默认保留规则

每组重复标签只保留一个，其余作为建议关闭项。默认保留规则按优先级执行：

1. 优先保留固定标签，因为固定标签通常是用户主动标记的长期页面。
2. 其次保留当前激活标签，因为用户正在查看它。
3. 最后保留最左侧标签，因为它最接近用户当前标签栏顺序。

如果同一组里同时存在多个固定标签，则保留最左侧固定标签。这个边界条件需要明确，因为固定标签都带有用户意图，不能随机选择。

### 确认流程

点击“智能去重”后，弹窗内展开确认区：

1. 插件扫描当前窗口标签。
2. 按重复组展示结果。
3. 每组显示页面标题、主域名、重复数量、判定原因和默认保留项。
4. 默认勾选建议关闭的重复标签。
5. 用户可以取消勾选某组重复项。
6. 点击“关闭已勾选重复标签”后才真正关闭标签。
7. 关闭完成后刷新弹窗状态，并提示实际关闭数量。

如果没有可关闭重复项，则显示“没有发现可关闭的重复标签”。这比静默无反馈更清楚，能让用户知道扫描已经完成。

## 工作集

### 命名

“会话”在界面中统一改名为“工作集”。内部存储可以继续沿用旧键名以兼容现有数据，但弹窗文案、状态提示和 README 后续应使用“工作集”。

### 保存工作集

点击“保存工作集”后：

1. 默认保存当前窗口所有标签。
2. 弹出名称输入，默认名称为 `YYYY-MM-DD HH:mm 的工作集`。
3. 用户确认后写入本地存储。
4. 保存成功后不自动关闭标签。
5. 状态区显示“已保存 N 个标签”，并提供一次性的“关闭已保存标签”按钮。

“关闭已保存标签”只关闭本次保存时记录的标签 ID。这样做是为了避免用户保存后又打开的新标签被误关。

### 工作集列表

工作集列表排序规则：

1. 收藏工作集优先展示。
2. 多个收藏工作集按收藏时间倒序展示。
3. 非收藏工作集按保存时间倒序展示。

每个工作集显示：

- 名称。
- 标签数量。
- 分组数量。
- 保存时间。
- 收藏状态。

列表数量继续沿用当前 `maxSessionCount`。第一版不做无限历史，因为本地存储容量有限，而且弹窗空间也不适合展示过长历史。

### 工作集操作

每个工作集支持以下操作：

- 恢复到当前窗口。
- 恢复到新窗口。
- 重命名。
- 删除。
- 收藏或取消收藏。

删除工作集前需要确认。原因是删除后无法从插件本地存储恢复，属于不可逆操作。

### 恢复规则

恢复到当前窗口时：

1. 按工作集保存的标签顺序创建新标签。
2. 跳过 Chrome 不允许扩展创建的内部页面。
3. 恢复后按主域名重建 Chrome 原生标签分组。
4. 尝试激活保存时的活跃标签，如果该标签无法恢复，则激活第一个成功恢复的标签。

恢复到新窗口时：

1. 先创建新窗口。
2. 将工作集标签创建到新窗口内。
3. 按主域名重建 Chrome 原生标签分组。
4. 尝试激活保存时的活跃标签。

恢复流程需要统计成功和失败数量。这样做是为了让用户知道是否有 `chrome://` 等内部页面被跳过。

## 数据结构

现有会话结构可以演进为工作集结构：

```js
{
  id: 'session-时间戳',
  name: '工作集名称',
  createdAt: 1780565295000,
  updatedAt: 1780565295000,
  favorite: false,
  favoritedAt: 0,
  activeUrl: 'https://example.com',
  tabs: [],
  groups: []
}
```

字段说明：

- `id`：工作集唯一标识，继续兼容旧的 `session-时间戳` 格式。
- `name`：工作集展示名称，用户可以重命名。
- `createdAt`：首次保存时间，用于展示和排序。
- `updatedAt`：最近修改时间，用于后续扩展更新工作集时使用。
- `favorite`：是否收藏，用于优先展示常用工作集。
- `favoritedAt`：收藏时间，用于多个收藏工作集之间排序。
- `activeUrl`：保存时的活跃标签地址，用于恢复后尽量回到用户当时的上下文。
- `tabs`：标签快照列表。
- `groups`：主域名分组快照列表。

读取旧数据时，如果缺少 `favorite`、`favoritedAt`、`updatedAt` 字段，需要自动补默认值。这样做是为了兼容已经保存过会话的用户。

## 消息接口

后台 service worker 需要支持以下动作：

- `scan-duplicates`：扫描当前窗口重复标签，返回重复组，不关闭标签。
- `close-selected-duplicates`：关闭用户确认的标签 ID 列表。
- `save-workspace`：保存当前窗口为工作集。
- `close-saved-tabs`：关闭本次保存的标签 ID 列表。
- `restore-workspace`：恢复工作集到当前窗口。
- `restore-workspace-new-window`：恢复工作集到新窗口。
- `rename-workspace`：重命名工作集。
- `delete-workspace`：删除工作集。
- `toggle-workspace-favorite`：收藏或取消收藏工作集。

可以保留旧动作名 `save-session` 和 `restore-session` 做兼容，但新界面和新代码路径应优先使用工作集语义。

## 错误处理

- 扫描重复失败时显示“扫描重复标签失败”。
- 关闭重复失败时显示实际关闭数量和失败数量。
- 保存工作集失败时显示“保存工作集失败”。
- 恢复工作集失败时显示“恢复工作集失败”。
- 删除工作集前必须二次确认。
- 重命名为空时不允许保存，因为空名称会降低列表可读性。
- 恢复时跳过内部页面，并在结果中计入失败数量。

## 校验与测试

需要覆盖以下场景：

- 完整网址重复可以被识别。
- 忽略追踪参数后的重复可以被识别。
- 路径不同或哈希不同的页面不会被错误合并。
- 固定标签优先保留。
- 激活标签优先保留。
- 没有固定或激活标签时保留最左侧标签。
- 用户取消勾选后，对应标签不会被关闭。
- 保存工作集后不会自动关闭标签。
- “关闭已保存标签”只关闭保存时记录的标签。
- 工作集可以重命名、删除、收藏和取消收藏。
- 收藏工作集排序优先于非收藏工作集。
- 工作集可以恢复到当前窗口。
- 工作集可以恢复到新窗口。
- 旧会话数据缺少新字段时可以正常展示。

## 后续扩展

本设计刻意不引入独立详情页，但消息接口和数据结构保留了迁移空间。后续如果弹窗空间不足，可以把智能去重确认区或工作集管理迁移到插件独立页面，而不需要重写后台核心逻辑。
