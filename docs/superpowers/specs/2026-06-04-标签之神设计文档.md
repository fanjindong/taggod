# 标签之神设计文档

## 背景与目标

`标签之神` 是一个 Chrome 插件，目标是解决用户同时打开上百个标签页时难以整理、难以定位、重复标签过多、临时工作现场难以保存的问题。

首版采用轻量弹窗形态，不引入构建工具，优先保证插件可以直接通过 Chrome 开发者模式加载。首版核心目标是让用户在一个弹窗内完成标签概览、搜索定位、一键整理、关闭重复标签和保存会话。

## 首版范围

首版包含以下能力：

- 整理标签页：固定标签优先，优先分组靠前，其余标签按域名排序，并使用 Chrome 原生标签分组能力创建分组。
- 关闭重复标签：按完整网址识别重复标签，保留当前正在使用或位置更靠左的标签。
- 搜索标签页：按标题、网址、域名搜索当前窗口标签，并支持点击切换。
- 保存当前会话：保存当前窗口标签列表、激活标签、分组快照和保存时间。
- 恢复最近会话：从本地存储恢复已保存会话，并尽量还原标签顺序与分组。
- 优先分组：通过星标分组让重要分组在每次整理后靠前，同时自动尊重当前标签栏里的分组顺序。
- 标签概览：展示当前窗口标签数、域名数、重复标签数和分组数。
- 快捷键：通过 Manifest V3 命令触发一键整理等高频操作。

首版不包含以下能力：

- 云同步。
- 跨设备同步。
- 复杂规则引擎。
- 多窗口工作台。
- 拖拽式批量管理页面。

这些能力会增加权限、交互和恢复逻辑复杂度，不适合放入首版。

## 技术结构

### `manifest.json`

使用 Manifest V3 配置插件基础信息、权限、弹窗入口、后台服务和快捷键命令。

需要声明的核心权限：

- `tabs`：用于查询、激活、关闭、移动和创建标签页。
- `tabGroups`：用于创建与更新 Chrome 原生标签分组。
- `storage`：用于保存会话、分组快照和用户配置。

### `popup.html / popup.css / popup.js`

弹窗层负责界面展示和轻量交互：

- 展示当前窗口标签概览。
- 展示搜索框和搜索结果列表。
- 展示一键整理、关闭重复、保存会话、恢复会话、优先分组等操作按钮。
- 发送消息给 `background.js` 执行批量操作。
- 在操作完成后刷新当前窗口标签状态。

弹窗层不直接承载复杂批量逻辑，原因是 MV3 弹窗生命周期较短，用户关闭弹窗后页面脚本会被销毁。批量操作集中放到后台服务中，可以减少中途状态丢失的风险。

### `background.js`

后台服务使用 MV3 `service worker`，负责所有有副作用的批量操作：

- 查询当前窗口标签。
- 按域名计算分组。
- 移动标签顺序。
- 调用 `chrome.tabGroups.group` 创建分组。
- 调用 `chrome.tabGroups.update` 设置分组标题和颜色。
- 关闭重复标签。
- 保存会话快照。
- 恢复会话并重建分组。
- 响应快捷键命令。

批量操作放在后台服务中，是为了让弹窗 UI 保持简单，并避免多个 UI 入口重复实现标签管理逻辑。

### `chrome.tabs`

用于完成标签页级别操作：

- `query`：获取当前窗口标签。
- `update`：激活用户点击的标签。
- `remove`：关闭重复标签。
- `move`：按域名重排标签顺序。
- `create`：恢复会话时重新打开标签。

涉及关闭标签时必须保守处理。默认只关闭完整网址一致的重复标签，避免把同站点不同页面误判为重复。

### `chrome.tabGroups`

用于管理 Chrome 原生标签分组：

- 按域名创建分组。
- 设置分组标题。
- 设置分组颜色。
- 恢复会话时尽量重建分组。

分组标题优先使用域名。域名为空或无法解析时使用 `其他`，原因是浏览器内部页面、扩展页面或异常网址可能无法得到标准域名。

### `chrome.storage.local`

用于保存本地状态：

- 会话快照：保存时间、标签列表、激活标签、标签顺序。
- 分组快照：域名、分组标题、颜色、标签数量。
- 用户配置：重复标签保留策略、会话保存数量上限、是否整理后自动分组。
- 优先分组配置：分组标识、分组标题、排序值和星标时间。

首版默认会话保存数量上限为 `10`。这个数字用于避免本地存储无限增长，同时保留用户最近多个工作现场。

## 数据模型

### 标签快照

```js
{
  id: 123,
  title: "页面标题",
  url: "https://example.com/path",
  favIconUrl: "https://example.com/favicon.ico",
  active: false,
  pinned: false,
  index: 0,
  groupKey: "example.com"
}
```

字段含义：

- `id`：保存时的标签页标识。恢复会话时不会复用旧值，因为 Chrome 会为新标签分配新标识。
- `title`：用于搜索和展示。
- `url`：用于去重、恢复和搜索。
- `favIconUrl`：用于增强列表识别度。
- `active`：用于保存当前工作焦点。
- `pinned`：用于恢复固定标签状态。
- `index`：用于恢复标签顺序。
- `groupKey`：用于按域名恢复分组。

### 会话快照

```js
{
  id: "session-时间戳",
  name: "2026-06-04 10:30 保存的会话",
  createdAt: 1780540200000,
  activeUrl: "https://example.com",
  tabs: [],
  groups: []
}
```

字段含义：

- `id`：会话唯一标识。
- `name`：展示给用户的会话名称。
- `createdAt`：用于排序和清理旧会话。
- `activeUrl`：恢复后用于尽量激活原来的工作页面。
- `tabs`：标签快照列表。
- `groups`：分组快照列表。

## 核心流程

### 整理标签页

1. 查询当前窗口全部标签。
2. 解析每个标签的域名，无法解析时归入 `其他`。
3. 读取用户保存的优先分组配置。
4. 按固定标签、星标状态、当前窗口分组顺序、域名、原始位置排序。
5. 调用 `chrome.tabs.move` 移动标签。
6. 对每个域名调用 `chrome.tabs.group` 创建分组。
7. 调用 `chrome.tabGroups.update` 设置标题和颜色。
8. 返回整理结果，包括处理标签数和创建分组数。

排序时保留固定标签优先，是因为固定标签通常代表用户长期入口，不应该被移动到普通标签之后。优先分组来自用户点星标，这样用户不需要手写域名列表。当前窗口分组顺序会作为普通分组的默认排序依据，原因是用户手动拖动分组后，插件应该尊重这个即时意图。

### 关闭重复标签

1. 查询当前窗口全部标签。
2. 以完整 `url` 建立映射。
3. 对同一 `url` 的标签优先保留当前激活标签。
4. 如果没有激活标签，则保留索引最小的标签。
5. 调用 `chrome.tabs.remove` 关闭其余重复标签。
6. 返回关闭数量。

只按完整网址去重，是为了避免误关同一网站下不同路径的页面。

### 保存会话

1. 查询当前窗口全部标签。
2. 生成标签快照。
3. 生成分组快照。
4. 读取已有会话。
5. 将新会话放在最前。
6. 超过保存上限时删除最旧会话。
7. 写入 `chrome.storage.local`。

### 恢复会话

1. 读取目标会话。
2. 按快照顺序调用 `chrome.tabs.create` 创建标签。
3. 恢复固定标签状态。
4. 按 `groupKey` 聚合新创建的标签。
5. 调用 `chrome.tabGroups` 重建分组。
6. 尽量激活原来的 `activeUrl`。

恢复时不关闭当前窗口已有标签，原因是首版要降低误删风险。用户可以在恢复完成后自行关闭不需要的标签。

## 异常处理

- 没有可管理标签时，弹窗显示空状态，不执行批量操作。
- 标签网址无法解析时，归入 `其他` 分组。
- Chrome 内部页面无法被恢复时，跳过该标签并在结果中统计失败数量。
- 批量关闭重复标签失败时，返回失败数量，避免整个操作静默失败。
- `chrome.storage.local` 写入失败时，弹窗显示失败提示。

## 测试策略

首版以手动验证和轻量脚本检查为主：

- 验证插件可以通过 Chrome 开发者模式加载。
- 验证弹窗可以展示当前窗口标签统计。
- 验证搜索可以按标题、网址、域名过滤。
- 验证一键整理后标签按域名靠拢，并创建原生标签分组。
- 验证关闭重复标签只关闭完整网址一致的页面。
- 验证保存会话后可以在弹窗看到最近会话。
- 验证恢复会话后标签顺序和分组尽量一致。
- 验证快捷键可以触发一键整理。

## 后续扩展方向

- 增加完整工作台页面，支持批量选择和更大规模的标签管理。
- 增加自动规则，例如超过指定标签数量后提示整理。
- 增加会话命名、收藏和删除。
- 增加按主题或关键词自动分组。
- 增加导出和导入会话能力。
