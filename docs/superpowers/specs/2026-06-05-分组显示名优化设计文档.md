# 分组显示名优化设计文档

## 目标

当前 Chrome 原生分组标题直接使用完整主域名，例如 `google.com`、`example.com.cn`。这类标题在标签栏里偏长，用户希望省略 `com` 等公共后缀，让分组名更短。

本次目标是优化“显示名”，不改变内部分组键。原因是 `foo.com`、`foo.net`、`foo.cn` 这类网站虽然短名相同，但实际主域名不同，如果把内部键也缩短会错误合并分组。

## 范围

- Chrome 原生分组标题使用短显示名，例如 `google.com` 显示为 `google`，`example.com.cn` 显示为 `example`。
- 弹窗“优先分组”和标签列表展示使用同一短显示名，降低界面噪音。
- 内部 `groupKey` 继续保存完整主域名，星标、排序、工作集保存和恢复仍用完整主域名。
- 当前窗口内如果短名冲突，则冲突项回退显示完整主域名，例如同时存在 `foo.com` 和 `foo.net` 时显示为 `foo.com`、`foo.net`。

## 显示名规则

1. `其他`、空值、内网短主机名、IP 地址保持原样，原因是它们没有可安全省略的公共后缀。
2. 普通二段主域名去掉最后一段，例如 `google.com` 显示为 `google`。
3. 常见多级公共后缀去掉后缀部分，例如 `example.com.cn` 显示为 `example`。
4. 如果多个完整主域名生成同一个短名，则这些主域名全部显示完整名称，避免用户看不出差异。

## 实现策略

新增两个纯函数：

- `getShortGroupTitle(groupKey)`：只负责把单个完整主域名转换为候选短名。
- `buildGroupTitleMap(groupKeys)`：基于当前窗口或恢复批次内的所有 `groupKey` 计算最终显示名，并处理短名冲突。

所有创建 Chrome 原生分组的地方都从 `buildGroupTitleMap` 取标题，避免整理当前窗口和恢复工作集显示不一致。

## 验证方式

- 校验 `google.com` 显示为 `google`。
- 校验 `example.com.cn` 显示为 `example`。
- 校验 `localhost`、`127.0.0.1`、`其他` 保持原样。
- 校验 `foo.com` 和 `foo.net` 同时出现时回退完整主域名。
- 运行 `node --check background.js`、`node --check popup.js` 和 `node scripts/校验插件.js`。
