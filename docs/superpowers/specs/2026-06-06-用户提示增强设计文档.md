# 用户提示增强设计文档

## 背景

`标签之神` 当前弹窗已经支持点击结果跳转、上下键选择、回车跳转、整理快捷键和工作集快捷键，但这些能力主要写在 README 和功能实现里。真实用户打开弹窗时，只能看到搜索框、结果列表和按钮，无法立即知道哪些地方可以点击、哪些键盘操作可用、整理是否有快捷键。

本次迭代目标是把关键操作提示放回用户正在操作的界面里，让用户不需要阅读文档也能发现高频能力。

## 目标

- 标题区用轻量键位样式提示默认打开弹窗快捷键，方便用户下次直接唤起插件。
- 搜索框附近用轻量键位提示说明“点击打开、上下键选择、回车跳转”。
- 整理入口用轻量键位样式提示默认快捷键，降低用户对高频操作的学习成本。
- 更多工具区只保留智能去重的风险短提示，避免低风险功能说明挤占界面。
- 保持快速切换首屏轻量，不引入教程弹窗或复杂帮助系统。
- 所有新增界面文案使用中文，避免英文提示混入。

## 非目标

- 不新增首次使用教程，原因是教程需要记忆状态和关闭逻辑，会增加弹窗复杂度。
- 不新增可配置帮助开关，原因是当前提示内容很短，常驻展示的维护成本更低。
- 不修改现有快捷键配置，原因是本次只解决提示不足，不改变命令行为。
- 不把 README 内容完整搬进弹窗，原因是弹窗空间有限，只展示当下有用的信息。

## 方案比较

### 方案一：常驻轻提示

在搜索框下方、整理按钮区域和更多工具小节标题附近放置短提示。提示始终可见，但字号和颜色弱于主要内容。

优点是用户第一次打开就能看到，不需要额外点击；实现简单，不需要存储状态。缺点是会占用少量弹窗高度。

### 方案二：首次教程浮层

第一次打开弹窗时展示引导浮层，说明点击、回车、上下键和快捷键。

优点是首次解释更集中。缺点是需要记录是否已读，也容易挡住用户真正想操作的搜索结果。

### 方案三：可折叠帮助区

在弹窗增加“帮助”入口，点击后展开完整操作说明。

优点是首屏更干净。缺点是用户不知道有能力时，也往往不会主动点帮助，因此不能解决“发现不了”的核心问题。

推荐采用方案一。原因是本次缺口不是说明不够完整，而是关键能力没有出现在操作现场；常驻轻提示能用最低复杂度覆盖这个问题。

## 界面设计

标题区在摘要下方增加默认打开快捷键提示：

```text
打开弹窗 ⌘⇧L / Ctrl Shift L
```

这个提示放在标题区，原因是它描述的是插件入口本身，不属于搜索框或整理按钮的局部操作。

搜索区在输入框下方增加轻量键位提示：

```text
点击打开 · ↑↓ 选择 · Enter 跳转
```

这里把“点击打开”放在最前面，原因是鼠标用户比键盘用户更需要被明确告知结果项可点击。键盘操作使用 `kbd` 样式弱化为辅助信息，避免搜索区变成说明段落。

整理按钮保留首屏位置，按钮文案仍为“整理当前窗口”。按钮下方增加短提示：

```text
默认快捷键 ⌘⇧Y / Ctrl Shift Y
```

这样做的原因是整理是高频动作，快捷键比点击按钮更适合频繁整理标签页；同时必须标明这是默认快捷键，避免和用户在 Chrome 里自定义快捷键后的实际配置冲突。

更多工具展开后，只在“管理操作”标题附近保留智能去重提示：

- 管理操作：提示智能去重会先确认，不会直接关闭。

只保留智能去重提示，原因是它涉及关闭标签的风险边界；优先分组和工作集属于低风险功能，常驻说明会让更多工具区显得像说明书。

## 结构设计

`popup.html` 负责新增静态提示节点。提示节点使用语义上靠近对应控件的位置，避免后续维护时文案和功能脱节。

`popup.css` 新增统一的轻提示和 `kbd` 样式。提示使用更浅的弱文本颜色、11px 字号和紧凑行高，让提示弱于标题和按钮。搜索区提示需要设置弹性布局，避免键位和说明在 420px 弹窗宽度下显得拥挤。

`popup.js` 不需要新增交互状态。快捷键提示使用默认快捷键短文案，不根据系统平台动态切换，原因是 Chrome 快捷键可以被用户自定义，静态提示只表达默认建议。

## 文案规则

- 提示必须短，不超过一行到两行。
- 提示只说明用户下一步能做什么，避免解释内部实现。
- 涉及破坏性操作时必须说明边界，例如去重不会直接关闭。
- 不使用英文句子；`Command`、`Shift`、`Ctrl`、`macOS`、`Windows` 和 `Linux` 属于快捷键与平台技术名词，可以保留。

## 错误处理

本次不新增错误状态。现有失败提示继续使用底部 `statusText`。新增提示不能替代操作结果提示，原因是用户执行整理、去重、保存后仍需要看到实际处理数量。

## 验证方式

- 打开弹窗后，标题区能看到默认打开弹窗快捷键提示。
- 打开弹窗后，搜索框附近能看到点击、上下键和回车提示，且提示是轻量键位样式。
- 搜索结果仍可点击跳转。
- 搜索框聚焦时，上下键仍能移动高亮结果。
- 搜索框聚焦时，回车仍能跳转到高亮结果。
- 首屏能看到整理按钮和默认整理快捷键提示。
- 展开更多工具后，能看到智能去重的风险短提示，不再为优先分组和工作集展示常驻说明。
- 新增文案全部为中文，技术名词除外。
- 运行 `node --check popup.js`。
- 运行 `node scripts/校验插件.js`。

## 验收标准

- 用户不读 README，也能从弹窗知道点击结果可以跳转。
- 用户不读 README，也能从弹窗知道上下键可以选择结果、回车可以跳转。
- 用户不读 README，也能从弹窗知道默认打开弹窗快捷键和整理当前窗口的默认快捷键。
- 新增提示不遮挡搜索结果，不让首屏主任务从“快速切换”变成“阅读说明”。
