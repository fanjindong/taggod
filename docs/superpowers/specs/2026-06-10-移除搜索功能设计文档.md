# 移除搜索功能设计文档

## 背景

Chrome 原生已经支持标签搜索、历史记录搜索和地址栏找回页面，本项目继续维护搜索入口会稀释定位。`标签之神` 的核心价值应收敛到“一键整理当前窗口”，护城河来自分组规则，而不是重复 Chrome 原生搜索能力。

## 目标

- 弹窗首屏只服务整理当前窗口，不再展示搜索框、搜索结果、最近使用列表和键盘跳转提示。
- 后台状态只读取当前窗口标签，不再为了搜索扫描所有窗口标签。
- 删除跨窗口激活标签和最近访问记录链路，减少非核心逻辑。
- README、校验脚本和界面文案同步强调“一键整理”和“分组规则”。

## 非目标

- 不改变一键整理排序规则。
- 不改变智能去重、工作集保存恢复、优先分组和分组阈值能力。
- 不新增整理预览视图，原因是本次目标是收敛产品定位，避免引入新的展示规则。

## 界面设计

弹窗标题改为“一键整理”，说明文案强调把当前窗口归拢成清晰分组。首屏保留标签概览、整理主按钮、快捷键提示和更多工具入口。更多工具中继续展示分组阈值、智能去重、优先分组和工作集。

## 后台设计

`getPopupState()` 只调用 `queryCurrentWindowTabs()` 和读取设置/工作集，不再调用 `queryAllWindowTabs()` 来构建搜索快照。重复标签概览仍保留全窗口扫描，因为它服务智能去重提示，和搜索能力无关。

删除 `recentAccess` 存储键、最近访问归一化函数、标签激活/关闭监听中的最近访问维护、`activate-tab` 消息入口和跨窗口激活函数。这样后台只保留整理、去重、工作集和分组规则相关逻辑。

## 校验

本地校验脚本需要改为验证搜索相关 UI 与文档文案不存在，并继续覆盖主域名归并、分组阈值、排序、去重和工作集行为。最终通过 `node scripts/校验插件.js` 验证。
