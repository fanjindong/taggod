# 搜索最近关闭标签页设计文档

## 背景

`标签之神` 当前搜索只覆盖所有已打开窗口里的标签页。Chrome 原生搜索可以命中最近关闭的历史标签页，用户在关闭页面后如果马上意识到还需要它，需要离开插件去浏览器历史或原生搜索里找回。

本次迭代在现有搜索体验上补齐“最近关闭标签页”搜索能力。功能只在用户输入搜索词后启用，空搜索仍保持现有“最近使用的已打开标签页”逻辑，避免默认列表被已经关闭的页面干扰。

## 目标

- 输入搜索词后，同时搜索已打开标签页和最近关闭标签页。
- 空搜索时不展示最近关闭标签页，只展示现有最近使用结果。
- 最近关闭结果需要有明确标识，避免用户误以为点击后只是切换到现有标签。
- 点击或回车选择最近关闭结果时，恢复对应标签页或窗口会话。
- 搜索输入过程不反复调用后台接口，避免每个按键都触发 Chrome API。
- 限制最近关闭结果数量和渲染数量，保证弹窗响应稳定。

## 非目标

- 不搜索全量浏览历史，原因是浏览历史权限更敏感，数据规模也远大于本次需求。
- 不在空搜索默认态展示最近关闭结果，原因是用户已经明确要求只有输入搜索词后才搜索历史关闭标签。
- 不把工作集纳入本次搜索范围，原因是工作集恢复是插件自有能力，和 Chrome 最近关闭会话属于不同来源。
- 不做历史结果的本地长期缓存，原因是 Chrome 已维护最近关闭会话，重复保存会增加隐私和过期一致性问题。

## 数据来源

后台新增读取 `chrome.sessions.getRecentlyClosed()`。该接口返回 Chrome 最近关闭的标签页或窗口会话，不是全量浏览历史。Chrome 对单次查询的 `maxResults` 限制为最多 25 个会话，因此后台只请求最近 25 个会话，再把关闭窗口里的标签拆分为可搜索结果。

扩展需要在 `manifest.json` 新增 `sessions` 权限。README 的权限说明同步增加中文解释：该权限只用于读取和恢复最近关闭的标签页或窗口，不上传标签数据。

最近关闭会话会转换为搜索快照：

- 单个关闭标签页转换为一条最近关闭搜索结果，恢复编号来自 `tab.sessionId`。
- 关闭窗口会拆分为多个标签页搜索结果，每个结果保留同一个 `window.sessionId`，点击任意结果时按 Chrome 原生粒度恢复整个窗口。
- 每条结果保留标题、网址、主域名、短分组名、关闭时间和来源类型。
- 无法解析网址时归入“其他”，保持和已打开标签页一致的兜底规则。

## 搜索行为

前台状态增加 `recentlyClosedTabs`，和现有 `tabs` 分开保存。

空搜索时，`getVisibleTabsFromState` 只使用 `state.tabs`，保持现有最近使用排序和数量限制。

输入搜索词后，搜索池由两部分组成：

1. 已打开标签页。
2. 最近关闭标签页。

两类结果使用同一套文本匹配字段和同一套排序规则：标题、网址、主域名和短分组名参与匹配评分，最近使用或关闭时间作为全局兜底排序依据。这样已打开标签和最近关闭标签的搜索体验保持一致。

搜索结果最多渲染 100 条。这个上限用于保护弹窗 DOM 数量，避免用户近期关闭过大窗口时输入卡顿。最近关闭快照最多纳入 100 个标签页，原因是 Chrome 最近关闭会话不是全量历史，超过 100 条后对“刚关闭找回”的收益降低。

## 结果展示

已打开结果保持现有展示：

- 标题。
- 分组或主域名。
- 网址。
- 当前窗口或其他窗口。
- 最近使用时间。

最近关闭结果增加明显标识：

- 元信息显示“最近关闭”。
- 如果有关闭时间，显示“最近关闭 · 3 分钟前”。
- 标题和网址仍按现有结构展示，保证视觉稳定。

搜索无结果时，空状态文案从“没有匹配的已打开标签页”调整为“没有匹配的标签页”，原因是搜索范围已经包含最近关闭结果。

## 恢复行为

结果项增加 `resultType`：

- `open`：已打开标签页，继续调用现有 `activate-tab`。
- `recentlyClosed`：最近关闭标签页，调用新增后台动作 `restore-closed-session`。

恢复最近关闭结果时，后台调用 `chrome.sessions.restore(sessionId)`。如果恢复的是单标签会话，浏览器会重新打开该标签；如果结果来自关闭窗口中的某个标签，Chrome 可能恢复整个窗口会话。这里沿用 Chrome 的会话恢复语义，不在插件层模拟只恢复窗口中的单个标签，原因是浏览器只保证按会话粒度恢复。

恢复成功后，弹窗展示“已恢复最近关闭标签页”，并刷新状态快照。恢复失败时展示“历史标签可能已过期”，原因是最近关闭会话可能已经被新的关闭记录挤出或被浏览器清理。

## 性能策略

后台在弹窗加载状态时读取一次已打开标签和最近关闭会话。前台输入变化时只在内存中过滤和排序，不向后台发送新的搜索请求。

具体限制：

- 最近关闭会话最多请求 25 条，这是 Chrome API 的硬性限制。
- 最近关闭标签快照最多 100 条，窗口会话拆分后达到上限即停止。
- 搜索结果渲染最多 100 条。
- 空搜索不处理最近关闭列表，避免默认态承担无用排序成本。
- 关闭窗口拆分标签时按 Chrome 返回顺序收集，达到上限后停止。

这些限制的目的是保证弹窗首开和输入过程都保持稳定，同时保留“刚关闭页面找回”的主要价值。

## 前后台职责

后台负责：

- 新增 `sessions` 权限后的最近关闭会话读取。
- 将最近关闭会话转换为前台可搜索快照。
- 执行 `restore-closed-session`。
- 处理恢复失败并返回稳定中文错误。

前台负责：

- 保存最近关闭快照。
- 只在有搜索词时合并最近关闭结果。
- 对不同结果类型调用不同动作。
- 渲染最近关闭标识。
- 保持键盘上下选择和回车行为一致。

## 错误处理

- 读取最近关闭会话失败时，不影响已打开标签搜索，并展示普通状态提示即可。
- 最近关闭快照为空时，搜索仍正常返回已打开标签页结果。
- 恢复会话失败时，提示“历史标签可能已过期”。
- 如果结果缺少 `sessionId`，前台不执行恢复并提示“历史标签无效”。
- 如果已打开标签在点击前已关闭，沿用现有“目标标签可能已关闭”提示。

## 验证方式

- 运行 `node --check background.js`。
- 运行 `node --check popup.js`。
- 运行 `node scripts/校验插件.js`。
- 手动验证空搜索时不展示最近关闭标签页。
- 手动验证输入关键词后可以看到最近关闭结果。
- 手动验证点击最近关闭结果可以恢复页面。
- 手动验证回车选中最近关闭结果可以恢复页面。
- 手动验证恢复失败时展示中文错误提示。

## 验收标准

- 未输入搜索词时，最近使用列表行为和现有版本一致。
- 输入搜索词后，已打开标签和最近关闭标签都能被搜索到。
- 最近关闭结果有清晰中文标识。
- 已打开结果点击后仍切换标签。
- 最近关闭结果点击后恢复标签页或会话。
- README 权限说明完整且为中文。
